// Who's Coming to the Party?

Features

Your vendors have vendors who have vendors. Time to find out who's really on the guest list.

search

Comprehensive DNS Analysis

SPF, DMARC, DKIM, MX, NS, CNAME, and domain-verification TXT records — the vendor fingerprints hiding in plain sight. Always on.

verified_user

Trust-Center Subprocessors

Finds and parses the vendor/subprocessor lists companies publish on their own trust centers — their guest list, in their words.

travel_explore

Runtime Web Traffic

Loads the site in headless Chrome and watches who it actually calls — third-party SDKs in page source plus live network requests.

workspace_premium

Certificate Transparency

CT-log discovery round-robined across crt.sh and Cert Spotter (plus MerkleMap and Censys with free keys), with automatic failover.

apartment

SaaS Tenants & Subdomains

Probes tenant subdomains (think acme.slack.com) and enumerates subdomains via CNAME discovery to catch what DNS records alone miss.

psychology

Offline Org Attribution

WHOIS/RDAP, a curated known-vendor dataset, and an embedded GLiNER NER model resolve domains to real organizations — no API keys, fully offline.

fact_check

Evidence on Every Edge

Each relationship carries the raw record, request, or URL it was inferred from — findings are auditable, not asserted.

bolt

Fast & Polite

Rust, async, cached, with batch scanning — and always-on DNS/HTTP/WHOIS rate limiters so a deep scan stays a good network citizen.

// The Main Event

How the Party Gets Started

From domain to full vendor map in seconds.

01

Send the Invites

Point it at a domain. DNS analysis runs first, every time — SPF includes, DMARC/DKIM, MX, NS, CNAME, and domain-verification tokens.

02

Sweep the Venue

Flip on the extended methods: trust-center subprocessor lists, Certificate Transparency logs, runtime web traffic in headless Chrome, SaaS tenant probes, and subdomain enumeration.

03

Check the Guest List

Every discovered domain resolves to a real organization: WHOIS/RDAP first, then a curated known-vendor dataset, then the offline NER model — with provenance tracked per attribution.

04

Follow the Plus-Ones

Each vendor gets the same treatment, recursively, to your chosen depth — and common infrastructure (AWS, Azure, Cloudflare) is bounced at the door so the graph stays meaningful instead of runaway.

05

Hand Out Party Favors

Export CSV, JSON, Markdown, or an interactive layered HTML vendor graph — with the raw evidence attached to every relationship.

Get the Party Started

Pick your platform — we've pre-selected yours.

laptop_mac

macOS

Grab the self-contained binary (embedded NER — works offline) for your chip and run it.

Apple Silicon

curl -L -o nthpartyfinder.tgz \
  https://github.com/grcengineering/nthpartyfinder/releases/latest/download/nthpartyfinder-aarch64-apple-darwin.tgz
tar -xzf nthpartyfinder.tgz && ./nthpartyfinder --help

Intel

curl -L -o nthpartyfinder.tgz \
  https://github.com/grcengineering/nthpartyfinder/releases/latest/download/nthpartyfinder-x86_64-apple-darwin.tgz
tar -xzf nthpartyfinder.tgz && ./nthpartyfinder --help

Every release ships a .sha256 checksum and signed SLSA build provenance. macOS includes the required whois client out of the box.

terminal

Linux (x86-64)

Self-contained binary with embedded NER — no runtime dependencies beyond a whois client.

curl -L -o nthpartyfinder.tgz \
  https://github.com/grcengineering/nthpartyfinder/releases/latest/download/nthpartyfinder-x86_64-unknown-linux-gnu.tgz
tar -xzf nthpartyfinder.tgz && ./nthpartyfinder --help

Need whois? apt install whois (or your distro's equivalent). Checksums + SLSA provenance ship with every release.

desktop_windows

Windows (x86-64)

PowerShell — curl.exe and tar are built into Windows 10+.

curl.exe -L -o nthpartyfinder.tgz `
  https://github.com/grcengineering/nthpartyfinder/releases/latest/download/nthpartyfinder-x86_64-pc-windows-msvc.tgz
tar -xzf nthpartyfinder.tgz
.\nthpartyfinder.exe --help

WHOIS lookups need a whois client — use WSL or SysInternals whois.

deployed_code

Docker

Pre-built image with embedded NER and whois included. Start scanning immediately, on any platform.

docker pull ghcr.io/grcengineering/nthpartyfinder:latest

docker run -v "$(pwd)/output:/output" ghcr.io/grcengineering/nthpartyfinder:latest \
  -d example.com -r 2 -f json -o /output/results
construction

Build from source

Clone, build, customize. Requires the Rust toolchain.

git clone https://github.com/grcengineering/nthpartyfinder.git
cd nthpartyfinder/nthpartyfinder
cargo build --release

The default build embeds the NER model (~175 MB binary); add --no-default-features for a ~15 MB slim build without it.
